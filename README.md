# BickerstaffEtAl_CommunPsycholStage1_2026
Repository for the manuscript, data and code related to the Communications Psychology Registered Report Stage 1 submission "Photoreceptor mechanisms underlying physiological responses to bright light in humans"

---

## Folder description

- Folder `code` contains all the R code needed to run the calibration of the light source, the modelling of the pilot data (OO-EMG), the sensitivity analysis, and the visualisations included in the report.
- Folder `R` contains R scripts for easily accessing project-wide variable and functions, as well as the preprocessing functions.
- Folder `renv` contains files necessary for the `renv` package for manage the R environment.
- Folder `data` contains all the necessary data to run all analyses and visualisations included in the report, i.e. processed pilot data and calibration measurements.
- Folder `outputs` contains all the outputs generated from the code and data mentioned above, including all figures included in the report, the OO-EMG model fit, the design matrix, and the sensitivity analysis results.
- Folder `visual_abstract` contains the visual abstract figure
- Folder `manuscript_stage1` contains the Stage 1 document (`.docx`)

## Requirements

- R version: **[4.4.0]**
- RStudio
- For generating counterbalanced stimulus sequences: the [`DeBruijn command-line software`](github.com/gkaguirrelab/DeBruijn).

All R package dependencies are managed using [`renv`](https://rstudio.github.io/renv/).

The code was tested on MacOS 15.7.3 using R 4.4.0, and on Windows 11 Enterprise 24H2 using R 4.4.0.

## Reproducing the analyses

### 0. External command-line dependency: DeBruijn

The stimulus sequence generation script requires the [`DeBruijn command-line software`](github.com/gkaguirrelab/DeBruijn).

This tool is not managed by `renv` and must be installed separately.

Clone and compile the software. After compiling the software, create a project-level `.Renviron` file containing the full path to the executable. For example, on macOS or Linux:

```bash
DEBRUIJN_EXE=/Users/yourname/DeBruijn/debruijn
```

or on Windows:

```bash
DEBRUIJN_EXE=C:/Users/yourname/DeBruijn/debruijn.exe
```

Restart R after creating or editing `.Renviron`.

### 1. Clone this repository

```bash
git clone https://github.com/tscnlab/BickerstaffEtAl_PCIRRStage1_2026.git
```

### 2. Open the R project

Open `BickerstaffEtAl_PCIRRStage1_2026.Rproj` in RStudio (or set the working directory to the project root).

### 3. Restore the R environment

```r
install.packages("renv")
renv::restore()
```

### 4. Run the `.Rmd` scripts

Running the code will recreate all `.html` files, and:

- The OO-EMG LMM fit object and model output
- The design matrix
- The sensitivity analysis results (running the script takes many hours)
- Figures in outputs/figures/

We recommend running the code in their numbered order, i.e. `1_modelling`, `2_calibration`, `3_power_analysis`, and `4_visualisation`, as some scripts depend on outputs generated by other scripts.
